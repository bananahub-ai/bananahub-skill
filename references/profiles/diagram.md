# Profile: 图表与信息图 (Diagram)

## 触发关键词

图表、流程图、架构图、信息图、示意图、思维导图、组织架构、时间线、对比图、数据可视化、diagram、flowchart、infographic、chart、wireframe、blueprint

## 补全维度

基础优化完成后，检查并按需补全以下维度（仅补全用户未提及的）：

### 组织方式
- 流程类 → flowchart with directional arrows, top-to-bottom / left-to-right layout
- 层级类 → hierarchical tree structure, clear parent-child relationships
- 对比类 → side-by-side comparison layout, clear visual separation
- 时间线 → horizontal or vertical timeline with chronological markers
- 未指定 → 根据内容推断最合适的组织方式

### 视觉主题
- 默认建议 clean minimalist design, professional appearance
- 可选：corporate style, technical blueprint, hand-drawn sketch style
- 保持信息优先，装饰性元素最少化

### 标签与文字
- 建议 clear sans-serif labels, readable font size
- 图表中的标签文字按用户语言保留（中文标签就用中文）
- 关键数据和标题应突出显示

### 配色
- 默认建议有限的、有区分度的配色方案
- professional color palette with 3-5 distinct colors for differentiation
- 避免过于鲜艳或艺术化的配色

## 示例优化

**用户输入**: 画一个微服务架构图

**基础优化结果**: Create a microservices architecture diagram

**增强后结果**: Create a microservices architecture diagram, clean minimalist design, hierarchical layout with clear service boundaries, directional arrows showing data flow, professional color palette with distinct colors for each service layer, clear sans-serif labels

## 注意事项

- **绝不添加艺术化修饰**（光影、氛围、景深等与图表无关）
- 图表中的文字内容需要准确性，不要编造用户没提到的标签
- **始终提醒用户：AI 生成的图表数据可能不准确，请务必核实事实内容**
- 复杂图表建议用专业工具（Mermaid、draw.io），Gemini 更适合概念性示意图
- 可以在 prompt 中引用真实世界概念、技术术语或已知架构模式（如 "microservices with API gateway pattern"），利用 Gemini 的知识库提升图表准确性
